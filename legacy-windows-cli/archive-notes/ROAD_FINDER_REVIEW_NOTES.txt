Claude Review Notes - 2026-05-28
--------------------------------

For Codex to fix in the next iteration.

TESTED: Ran all 8 JPEG screenshots through the consolidated build.
Result: 56 clusters found at tolerance 5. BAT runs clean. No crashes.

BUGS TO FIX:

1. Black highlight uses raw_mask instead of filtered_mask.

   process_image() calls save_black_highlight(black_file, raw_mask, ...)
   but should use the cluster-filtered mask returned by detect_clusters().
   Same issue for save_overlay and save_transparent: all three should use
   filtered_mask so tiny noise pixels below min_cluster_size are excluded from
   the output images.

   Fix: change the output image calls from raw_mask to filtered_mask. The
   variable is already available:

   filtered_mask, clusters = detect_clusters(...)

2. CSV generation was removed but should be restored.

   The combined CSV (all_clusters_summary.csv) and per-image CSVs are useful
   for sorting clusters by pixel count and finding exact coordinates. The
   script currently prints "CSV files were not created", which is confusing.
   Either restore CSV or remove that message.

3. Grid mode was removed from the BAT per user request, but the Python script
   still supports --grid. This is fine: leave the CLI option in the Python
   script for advanced users but keep it out of the BAT menu.

UX IMPROVEMENTS TO ADD:

4. Auto-open report after completion.

   Add this before goto end_ok in the BAT:

   if exist "%OUTPUT_DIR%\report.html" start "" "%OUTPUT_DIR%\report.html"

5. Restore the cluster coordinate table in the HTML report.

   The earlier version had a per-record table showing cluster_id, pixel_count,
   center_x, center_y, bounding box, and dimensions. This was removed. Users
   need coordinates to find spots on the map.

6. Clear all old output files before each run, not just CSVs.

   Old PNG files from a previous tolerance run linger in output/ and confuse
   users who think they are from the current run.

   Suggestion: delete output/*.png, output/*.csv, and output/report.html at
   the start of each run.

OVERALL: The consolidated build is solid. The BAT is well-structured with
goto-based error handling, smart JPEG defaults, and clean escaping. The Python
script is cleaner than the original with proper validation and dataclass-based
output records.

Combine the best parts of the Claude and Codex versions into one portable

Important current decision:
Grid mode was intentionally removed to keep the workflow simpler. Please do
not re-add it unless you find a strong reason. Instead, check that no BAT

the generated images because screenshot coordinates are not useful to match
in-game.

Overlay, black highlight, and transparent mask files should use the raw
detected mask so even very small red hits remain visible. Any cluster counting
in the console/report is only secondary and must not hide pixels in the images.

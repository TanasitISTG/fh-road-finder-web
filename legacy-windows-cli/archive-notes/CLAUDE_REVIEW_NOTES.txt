BUGS TO FIX:

1. Black highlight uses raw_mask instead of filtered_mask (line ~382)
   process_image() calls save_black_highlight(black_file, raw_mask, ...)
   but should use the cluster-filtered mask returned by detect_clusters().
   Same issue for save_overlay and save_transparent — all three should
   use filtered_mask so tiny noise pixels below min_cluster_size are
   excluded from the output images.

   Fix: change lines 382, 387, 392 from raw_mask to filtered_mask.
   The variable is already available: _, clusters = detect_clusters(...)
   should be: filtered_mask, clusters = detect_clusters(...)

2. CSV generation was removed but should be restored.
   The combined CSV (all_clusters_summary.csv) and per-image CSVs
   are useful for sorting clusters by pixel count and finding exact
   coordinates. The script currently prints "CSV files were not created"
   which is confusing. Either restore CSV or remove that message.

3. Grid mode was removed from the BAT (per user request) but the Python
   script still supports --grid. This is fine — leave the CLI option in
   the Python script for advanced users but keep it out of the BAT menu.

UX IMPROVEMENTS TO ADD:

4. Auto-open report after completion.
   Add this before goto end_ok in the BAT:
     if exist "%OUTPUT_DIR%\report.html" start "" "%OUTPUT_DIR%\report.html"

5. Restore the cluster coordinate table in the HTML report.
   The earlier version had a per-record table showing cluster_id,
   pixel_count, center_x, center_y, bounding box, and dimensions.
   This was removed. Users need coordinates to find spots on the map.

6. Clear ALL old output files before each run, not just CSVs.
   Old PNG files from a previous tolerance run linger in output/ and
   confuse users who think they are from the current run.
   Suggest: delete output/*.png, output/*.csv, output/report.html
   at the start of each run.

OVERALL: The consolidated build is solid. The BAT is well-structured
with goto-based error handling, smart JPEG defaults, and clean escaping.
The Python script is cleaner than the original with proper validation
and dataclass-based output records.
